# BRIEF：task-watcher 可靠性优化（第一批）

## 功能名称和目标
功能名称：task-watcher 可靠性优化

目标：
- 让 task-watcher 在长时间运行中更稳定，不再因卡死或退出导致自动流转中断
- 降低超时误报，减少对 PM 和飞书的噪音干扰
- 为后续通知重试、去重和 watcher 演进打下可靠基础

## 本轮范围（林总工已确认）
本轮只做第一批高优先级项：
1. **进程级可靠性**：heartbeat + watchdog / 自动重启
2. **日志持久化**：重要事件写文件，保留 7 天
3. **超时检测优化**：区分“无响应”和“工作中”，减少误报

## 验收标准
1. watcher 主循环会持续更新 heartbeat 文件
2. watcher 异常退出或卡住后可被外层机制自动拉起或重启
3. 重要事件（状态流转、自动派发、重发、错误）会写入持久化日志
4. 超时策略改为：
   - 无 ack 且无 Working：超过 3 分钟才重发
   - Working 超过 30 分钟：通知 PM 介入，不重发
5. 不破坏现有 result/review/QA/close-task 自动流转主链路

## 边界和约束（什么不做）
- 本轮**不做**通知去重改造
- 本轮**不做**通知失败重试
- 本轮**不做**轮询效率优化（如 fswatch/inotify）
- 本轮优先最小可行改动，不重构整个 task-watcher 架构

## 优先级和关联功能
优先级：P0

关联功能：
- 事件驱动自动流转（result → review → QA → close）
- dev 自动认领
- arch-1 多 domain 并行派发
- send-to-agent.sh 消息投递
