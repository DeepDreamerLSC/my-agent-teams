# 任务：排查智谱模型联网搜索生产失效

## 目标
对 chiralium 生产环境中“智谱（GLM/Zhipu）模型联网搜索全部失效”做**只读诊断**，明确根因归类，并给出后续修复建议。

## 背景
- 用户反馈：生产环境里，智谱相关模型的联网搜索都失效了。
- 这是 **production issue**，当前阶段先做**事实排查与根因定位**，不要直接改代码、不要部署。
- 项目路径：`~/Desktop/work/chiralium`
- 生产域名：`https://chiraliumai.cn`

## 排查范围
请优先从以下四类可能性归因：
1. **前端 gating / 能力展示异常**
   - 按钮被错误置灰
   - capability / runtime status 展示错误
2. **后端联网搜索链路异常**
   - 请求已发起但 provider 参数错误
   - runtime route / tool-call / provider-native 搜索链路回退或失配
3. **生产配置漂移**
   - 智谱联网搜索相关环境变量缺失、值错误、与代码契约不一致
4. **外部依赖 / 上游退化**
   - 请求正确发出，但智谱侧能力不可用或返回异常

## 建议排查点
请尽量基于事实验证，不要只看代码猜测：
- 生产环境相关接口返回：
  - `/api/meta/chat-capabilities`
  - `/api/models/available`
  - 任何与 Zhipu / GLM capability / runtime status 相关接口
- 前端是否依据这些字段正确渲染
- 后端 Zhipu / GLM 联网搜索链路的当前实现
- 最近相关提交是否引入回退/回归
- 如果是配置问题，请明确：
  - 缺哪个变量
  - 应该填什么形态的值
  - 为什么当前值会导致联网搜索失效

## 交付要求
在 `result.json` 中至少写清楚：
1. **根因结论**（归属上面哪一类，允许多因一果）
2. **证据**（接口返回、日志、代码位置、配置项等）
3. **影响范围**（哪些模型/页面/链路受影响）
4. **是否代码问题** / **是否配置问题** / **是否上游问题**
5. **推荐后续动作**（拆成可执行修复任务）

## 约束
- 本任务以只读诊断为主，**不要直接改代码，不要部署**。
- 如确认为代码问题，只需指出最小修复面和涉及文件。
- 如确认为配置问题，只需给出需要调整的配置项，不要擅自变更生产。
