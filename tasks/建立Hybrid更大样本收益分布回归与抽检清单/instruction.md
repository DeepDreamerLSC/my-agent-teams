# 任务：建立 Hybrid 更大样本收益分布回归与抽检清单，量化扩样收益与覆盖边界

## 任务类型
verification

## 目标
使用现有 `regression-summary` 基础设施与已归档产物，补一轮**更大样本口径的收益分布回归与人工抽检清单**，把“当前收益发生在哪些样例/页型/来源、哪些空白仍需补样”清楚量化出来，供后续继续扩样与 owner 校对。

## 任务边界
- 只做回归执行、清单整理与报告归档，不改业务代码。
- 优先复用现有五样例 final-archive / hybrid-e2e / phase3-paddle-quality 产物，不额外要求本轮先补新模型能力。
- 如果本地真实 PDF 样本仍不足以构成严格意义上的“更大文档集”，则必须诚实转成“现有归档样本 + 页型/来源维度扩展”的分布回归，并显式列出缺失样本清单。

## 输入事实
- 当前 `artifacts/pdf2word/p2-regression/current-five-sample-baseline/` 已有五样例基线摘要。
- 当前下一阶段关注点已从“能不能跑通”转为“收益分布是否稳定、扩样清单是否明确”。
- 现有可复用产物至少包括：final-archive、hybrid-e2e-validation、phase3-paddle-quality 等。

## 约束
- `write_scope` 以 `task.json` 为准。
- 报告必须明确区分：真实已覆盖范围 vs 仍待补样的空白，不得把 5 个文档误写成“大样本已充分覆盖”。
- 抽检清单要能指导下一轮补样：至少包含建议优先级、页型/学科/答案/图片表格等缺口维度。

## 交付物
1. `artifacts/pdf2word/p2-regression/` 下新增一轮收益分布回归产物，至少包含：
   - manifest / summary json
   - markdown 说明
   - 抽检清单或补样清单
2. `result.json`：说明本轮采用的样本口径、收益分布结论、当前仍缺的样本维度。

## 验收标准
1. 产物能说明当前收益集中在哪些样例/页型/来源，而不是只重复五样例总数。
2. 报告中明确列出仍需补样的维度与优先级。
3. 结论与已有 artifacts 事实一致，不放大覆盖范围。
4. 结果可直接作为下一轮补样/抽检任务的输入。

## 下游动作
完成后进入 review-1 审查；通过后作为更大样本扩样与常态化抽检的底稿，再决定是否继续拆“新增真实样本补齐”执行任务。
